## Why

Phase 4 完成了 Vue 3 前端项目搭建（路由、布局、API 层、类型定义、ECharts 注册），但 5 个页面视图（Dashboard、费用基线、影响因素、成本估算、报告）目前仍为占位符。本阶段将实现所有页面的完整 UI 与数据交互，使平台从"骨架"变为"可用产品"——用户可查看 KPI、分析费用基线、探索影响因素、模拟成本估算、生成下载报告。

## What Changes

- **Dashboard 页面**：实现 KPI 卡片（总单量、总工时、月度费用、人效）、日出库单量趋势折线图、月度费用构成柱状图、操作类型工作量分布饼图
- **费用基线分析页面**：实现筛选面板（仓库、月份、费用类型）、月度基线数据表格、劳务单价对比柱状图、双仓月度对比图
- **影响因素分析页面**：实现因素重要性排序柱状图、Pearson 相关系数热力图、单因素散点图（可选因素 vs 工时）、双仓因素对比
- **成本估算/报价模拟页面**：实现参数输入表单（单量、件单比、人效、天数等）、实时计算结果展示（人数、工时、费用、单均/件均成本）、与历史数据对比
- **报告页面**：实现报告列表、报告生成表单（选仓库/时间范围）、Markdown 在线渲染预览、报告下载（HTML 格式）

## Capabilities

### New Capabilities

- `dashboard-page`: Dashboard 数据看板页面——KPI 卡片、趋势折线图、费用构成柱状图、工作量分布图
- `baseline-page`: 费用基线分析页面——筛选面板、月度基线表格、劳务单价对比图、双仓对比图
- `impact-page`: 影响因素分析页面——因素排序柱状图、相关系数热力图、散点图、双仓对比
- `estimate-page`: 成本估算/报价模拟页面——参数表单、实时结果、历史对比
- `report-page`: 报告管理页面——列表、生成表单、Markdown 预览、HTML 下载

### Modified Capabilities

（无需修改现有 spec 级别的行为要求）

## Impact

- **前端 `src/views/`**：替换 5 个占位符视图为完整页面实现
- **前端 `src/components/`**（新建）：抽取可复用图表组件和表单组件
- **API 调用**：所有 `src/api/index.ts` 中已定义的 12 个 API 方法将被页面消费
- **Pinia Store**：`useAppStore().currentWarehouse` 将驱动各页面的数据加载
- **依赖新增**：需要 `markdown-it`（Markdown 渲染）用于报告预览
- **后端/分析服务无变更**：本阶段仅前端 (frontend) 模块变更

## 非目标

- 不涉及后端 API 或 Python 分析服务的修改
- 不涉及数据库结构变更
- 不做移动端响应式适配（Phase 7 范畴）
- 不做用户认证/权限控制
